# プラットフォーム別ルール

## X

- **動画はファイルを直接添付**する（ネイティブ投稿）。YouTubeリンクの貼り付けは、リンクカードになって自動再生されず、外部リンクとしてリーチも下がるので使わない
- ハッシュタグは0〜1個。盛るとスパムっぽく見え、今のXでは検索効果も薄い
- 無音自動再生されるため、動画はテロップ焼き込み済みであること

## Facebook

- 動画はファイルを直接添付（Xと同じ理由）
- ハッシュタグは付けない

## YouTube Shorts

- 縦動画・3分以内なら自動的にShorts扱い。「#Shorts」タグは不要
- **タイトルは検索向け**に書く。
- **説明文にアプリ名を必ず入れる**（検索対象になる）。App Storeリンクも書く（タップ不可の場合があるが書く価値はある）
- 確実なリンク導線として**チャンネルプロフィールにApp Storeリンクを設定**しておく
- ハッシュタグは2〜3個。**最初の3個だけがタイトル上に表示される**ので見せたい順に置く
- **視聴者層は「いいえ、子ども向けではありません」を選ぶ**（親向けの紹介動画のため。「子ども向け」にするとコメント欄が無効になる）
- **「動画の言語」を設定する**（詳細画面→「すべて表示」→「言語とキャプションの認定」）。日本語版は日本語、英語版は英語
- 言語別に動画を分ける場合は、それぞれ別動画としてアップロードする（テロップ焼き込み・音声が言語ごとに違うため）
- サムネイル: Shortsはカスタムサムネイル不可・フィード自動再生でほぼ見られないので気にしない
- 再生リスト: 任意。アプリごとに作っておくと後で整理しやすい程度

## App Storeリンクの国コード

- URLは `apps.apple.com/【国コード】/app/...` の形式で、国コードが表示言語を決める
- 日本語向け投稿 → `/jp/`、英語向け投稿 → `/us/`
- 検索で拾ったURLは海外ストア（例: `/kw/`）のことがあるので必ず確認する
- iPhoneでは国コードに関係なく本人のストアが開くため、実害より見栄えの問題

## 文言の制約（仕様との整合・最重要）

- 投稿前に対象アプリのREADME / docsとエージェントメモリのアプリ別制約を確認する
- 「無料」→ 課金機能の追加予定があるアプリでは使わない
- 「オフライン」「完全オフライン」→ オンライン機能（音声認識・広告など）を使うアプリでは使わない
